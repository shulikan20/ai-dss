The matching pipeline

      
1
Domain filter. Capabilities outside the selected business domains are removed before any scoring.

      
2
SBERT retrieval. The bottleneck text is embedded and compared against pre-computed capability embeddings.

      
3
Dimension scoring. Five dimensions per candidate: semantic fit, pain-point match, data readiness, tech fit, integration compatibility.

      
★
Hybrid interception. SBERT's top-15 go to phi4, which reads the actual bottleneck text and scores each candidate's relevance 0–10.

      
4
TOPSIS ranking. Multi-criteria decision analysis over the five weighted dimensions produces the final order.

      
5
Explanations. The top 3 get contextual 2-sentence explanations.


      
TOPSIS weights

      

        
DimensionWeightPrimary input

        
          
Semantic fit0.35Bottleneck text similarity (phi4 score in hybrid)

          
Pain-point match0.30Confirmed + answer-derived pain flags

          
Data readiness0.20Structured exports, history months

          
Tech fit0.10Team technical level (1–3)

          
Integration compat0.05Existing platform stack

Export analyser

      
C
Classical: format detection, value inference, column-type classification.

      
L
LLM: column mapping to canonical fields and signal extraction.

      
H
Hybrid: classical structure plus LLM enrichment produces metrics, an enrichment summary, and suggested pain flags. Applied insights update the profile before matching.
